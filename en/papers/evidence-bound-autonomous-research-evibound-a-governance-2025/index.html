Evidence-Bound Autonomous Research (EviBound): A Governance Framework for Eliminating False Claims

arXiv 2025 60.6 method

TLDR

EviBound is a dual-gate governance framework that eliminates false claims in LLM-based autonomous research agents by requiring machine-checkable evidence.

Reasoning

The paper presents a clear, well-defined framework with empirical evaluation on benchmark tasks, showing significant reduction in hallucination rates. Strengths include concrete methodology and low overhead; weaknesses include limited task diversity and lack of real-world deployment evidence.

Deep Analysis

Innovations

  • Dual governance gates (pre-execution Approval Gate and post-execution Verification Gate) requiring machine-checkable evidence to eliminate false claims
  • Bounded, confidence-gated retries (1-2 attempts) to recover from transient failures without unbounded loops
  • Evidence-bound claim propagation: claims only propagate when backed by queryable run ID, required artifacts, and FINISHED status

Methodology

EviBound uses a pre-execution Approval Gate to validate acceptance criteria schemas before code runs, and a post-execution Verification Gate that validates artifacts via MLflow API queries with recursive path checking and optionally validates metrics. Claims propagate only with a queryable run ID, required artifacts, and FINISHED status. Bounded retries handle transient failures. Evaluated on 8 benchmark tasks against Prompt-Level Only and Verification-Only baselines, measuring hallucination rate and execution overhead.

Key Results

Baseline A had 100% hallucination (0/8 verified), Baseline B had 25% hallucination (2/8 fail verification), while EviBound achieved 0% hallucination (7/8 verified, 1 correctly blocked at approval gate) with ~8.3% execution overhead.
