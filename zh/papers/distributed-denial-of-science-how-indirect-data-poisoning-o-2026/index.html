深度分析

创新点

  • 间接数据投毒攻击：破坏开放数据集，误导自主研究代理生成欺诈性科学结论，无需直接访问或触发词。
  • 缓解措施：为代理设置科学家角色（scientist persona）。
  • 缓解措施：包含五项检查的数据来源审计（引用论文、社会标记、统计异常、相关数据集、投毒警告）。

方法

该研究在五个社会热点话题上评估了间接数据投毒攻击，使用了三种前沿AI系统（Claude Code with Claude Opus 4.7, Codex with GPT-5.5, Gemini CLI with Gemini 3.1 Pro）和450次符合伦理的实验运行。它测量了攻击成功率和检测率，并测试了两种防御措施：科学家角色和数据来源审计。

关键结果

投毒在49.56%的运行中成功，检测率仅为6.0%。科学家角色减少了但未消除投毒（成功率16.67%），而来源审计将攻击成功率降至零。
